Handcrab Documentation

Introduction

Handcrab is a wrapper around pandoc that automates some of the HTML formatting. Currently, handcrab can automate the styling of row headers and row/column headers in tables (column headers are done in pandoc by default), and can fill in the alt text for images. It can also create the description box with the hide/reveal button in case the alt tag isn’t big enough to hold the description. Handcrab comes with the --help or -h tags in order to view the passable arguments. Those arguments are as follows:

Argument name Short-form Is optional Description
--help -h Yes Displays the argument help message
--input -i No Pass the path to the LaTeX file
--output -o No Pass the path to the output HTML file
--resource-template -rt Yes Overrides path to resource template
--save-pandoc -s Yes Saves the raw pandoc output to a provided path
--version Yes Displays current version number
--update Yes Updates to current version of handcrab

For example, if your input file was Folder/MyInput.tex and you wanted to write the HTML Folder2/MyOutput.html you could run the following command:

handcrab -i "Folder/MyInput.tex" -o "Folder2/MyOutput.html"

If you wanted to specify a resource template, say Templates/MyTemplate.html, you could run the following

handcrab -i "Folder/MyInput.tex" -o "Folder2/MyOutput.html" -rt "Templates/MyTemplate.html"

The original handcrab script was created so that I wouldn’t have to copy the body of my pandoc output into the resource template every time I had compiled. Since I generally ran pandoc from the main folder, this script also requires that the file ResourceTemplate.html be in a specified location. Without passing the --resource-template argument to handcrab, it assumes that the path to the resource template is HTML/Files/ResourceTemplate.html. The --save-pandoc argument will create a file in the given path that only holds the unmodified pandoc output.

Handcrab runs the following command:

pandoc -f latex --mathjax -t html PATH_TO_INPUT -s -o TEMP_PATH --metadata title="..."

where PATH_TO_OUTPUT is the input path specified in the --input argument and TEMP_PATH is either the default or the path specified with the --save-pandoc argument.

Installation

  1. To install handcrab, navigate to the Github page, click the "Code" dropdown box, and click "Download Zip" at the bottom.

  2. Extract the folder to your desktop, and rename it to handcrab exactly as shown — all lowercase.
    Note: it is required to be on your desktop, as I needed to circumvent the fact that we don't have admin privileges

  3. Open a terminal (I use iterm) and navigate to the handcrab folder.

  4. Type in python3 setup.py

  5. Restart your terminal

Handcrab should now be fully installed. If it is not working, your terminal will probably tell you to run some kind of command when it starts up. Run that. It'll probably ask for your password, this is fine. The password won't display as you're typing it, but it is being read.

Resource Templates

The code reads the resource template, finds specific lines, and replaces them with their appropriate values. This was done horribly and in such a way where it really only works with the first resource template. In particular, it looks for the following two tags:

  1. <h1>Title</h1>

  2. <p>Content</p>

If the input file has an h1 header in it (I.E. \section{...}), then handcrab should ideally replace i. with whatever is in the section. I am not confident that this works. If it doesn't, let me know.

The modified body from handcrab's output replaces ii.

In other words. If those tags aren't in the templates, then it will not copy over the relevant information. If more than those tags are in the template, than the excess will also be copied over. The point of the templates was to copy over the preamble and any relevant formatting. I, however, went the lazy route during the implementation so now everyone's stuck with this.

Table Environments

The following sections describe how to add additional formatting to HTML tables from the LaTeX

Row Table Environment

To write a table where the first entry in each row is a header, you surround the tabular environment with four commas, and add !ROWTABLE! after the first four commas. The following example has been provided.

,,,,!ROWTABLE!
\begin{tabular}{|c|c|c|c|c|c|}
Digit & A & $5$ & $7$ & B& F\\
Decimal Digit & $10$ & $5$ & $7$ & $11$ & $15$\\
\hline
Exponent Value & $16^4$ & $16^3$ & $16^2$ & $16^1$ & $16^0$ \\
\hline
Numerical Value & $65536$ & $4096$ & $256$ & $16$ & $1$\\
\hline
,,,,

Handcrab works by modifying the HTML that’s produced by pandoc, so it is important that in the !ROWTABLE! environment, that the first row is not treated as a header, but just as plain data. The same principle follows for the !ROWCOLTABLE! environment discussed below.

Row Column Table Environment

To write a table where the cells in the first row and first column (excluding the top-left cell) are all treated as headers, you use the !ROWCOLTABLE! environment. Similar to the last one, you surround the table with four commas and specify the table type. An example is provided below.

,,,,!ROWCOLTABLE!
\begin{tabular}{|c|c|c|}
& Even Number & Odd Number \\ 
Multiple of 3  & 12, 150, 30, 72 & 61, 15, 21 \\ \hline
Not a multiple of 3 & 56, 88 & 61, 55 \\ \hline
\end{tabular}
,,,,

Once again, it’s important not to use the \hline command in the table as it will change the way pandoc outputs the table, and will break handcrab.

Image Descriptions

The following sections describe how to modify the alt attribute to the HTML image tag from the LaTeX.

The Altmarker Environment

Handcrab lets you specify alt text for images in a similar way. The main difference is that instead of placing the marker after the includegraphic command. An example is provided below.

,,,,
\includegraphics[width=0.35\textwidth]{unitcirc.png}
!ALTMARKER! Unit circle drawn on x y axis with the point (0,0) marked.
,,,,

The biggest thing to note is that the space after !ALTMARKER! is required to be there, even if the alt text is blank. Alternatively, you can replace !ALTMARKER! with !NOALT! to ensure a blank alt text. However, I’ve tested that once and then promptly forgot about it so I can’t remember if it actually works.

Long Descriptions

Sometimes the alt text isn’t big enough to hold the entire description, or sometimes you want extra formatting within the alt text, such as a table or a list. Handcrab has automated the creation of long descriptions. Unlike the previous environment, this one uses semicolons. This is because it allows for the table formatting to work within the long description. The long description environment encases a center environment (since that produces a div). It begins with 4 semicolons, followed by LONGDESC and the text that displays on your button, then a center environment containing your description, and finally another set of 4 semicolons. An example is provided below.

\begin{center}
,,,,
\includegraphics[width=0.35\textwidth]{hohmann.png}
!ALTMARKER! A very fancy image.
A description of the image is given in the following table.
,,,,
\end{center}

;;;;LONGDESC Hide/Reveal description of image
\begin{center}
,,,,!ROWCOLTABLE!
\begin{tabular}{|c|c|c|}
 & $\in 2\mathbb{Z}$ & $\not\in 2\mathbb{Z}$ \\
$\in 3\mathbb{Z}$ & 6 & 3 \\
$\not\in 3\mathbb{Z}$ & 4 & 7
\end{tabular}
,,,,
\end{center}
;;;;

It should be noted that the table does not count as a valid description to the image, but is just there to display that other environments work in the long description environment.
